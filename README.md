## 📖 为什么会有 GitHub-Trail？

在 GitHub 的浩瀚星空中，很多初学者（小白宝宝们）常常会遇到这些困扰：
- **“昨天看那个宝藏项目叫什么来着？”** —— 忘了 Star，历史记录又太难翻。
- **“我这一周到底学了啥？”** —— 感觉自己很忙，但没有直观的反馈。
- **“数据存在哪才安全？”** —— 既想同步多端，又担心隐私泄露。

**GitHub-Trail** 就是为了解决这些问题而生的。它像一个隐形的“数字化小助手”，默默记录你的成长轨迹，并用 AI 为你复盘。

---

## ✨ 核心能力

### 1. 🕒 智能足迹追踪 (Smart History)
* **自动记录**：无需额外操作，自动捕捉你访问过的仓库、Issue 和 Pull Request。
* **语义化分类**：自动识别你是去“看代码”、“提问题”还是“看文档”。
* **极致轻量**：毫秒级响应，完全不影响 GitHub 的原生访问体验。

### 2. ☁️ WebDAV 多端同步 (Privacy Sync)
* **数据确权**：支持坚果云、Nextcloud 等 WebDAV 服务，数据存放在你自己的云盘里。
* **无缝衔接**：宿舍刷完 GitHub，实验室电脑自动拉取最新历史。
* **隐私无忧**：脚本不经过任何第三方服务器，只在你的浏览器和你的网盘之间通信。

### 3. 💾 灵活导入导出 (Data Freedom)
* **一键搬家**：支持 JSON 格式数据导入导出。
* **定期备份**：养成备份习惯，让你的技术成长史永不丢失。

### 4. 🤖 AI 智能深度分析 (AI Insight)
* **技术画像**：基于你的浏览历史，AI 会自动总结你的技术偏好（如：倾向于前端、深度学习或后端工程）。
* **成长周报**：一键生成超酷的“技术学习周报”，清晰看到自己的进步。
* **避坑指南**：AI 会根据你查阅的 Issue，提醒你该领域常见的技术坑点。

## 🚀 快速上手

### 第一步：环境准备
确保你的浏览器已安装 **Tampermonkey (油猴)** 插件。

### 第二步：安装脚本
[点击此处安装 GitHub-Trail 脚本](#) *(这里替换为你的 GreasyFork 或脚本链接)*

### 第三步：初始化配置
1. 打开任意 GitHub 页面。
2. 找到右上角新增的 **🐾 Trail** 图标。
3. 如果需要同步，请在“设置”中填入你的 **WebDAV 地址**和**授权码**。
